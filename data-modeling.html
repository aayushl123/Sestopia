Relational Database Design

  • Skill Classification:Hard Technical Skill
  • Related Software Engineering Area(s) : Software Design

A Database is a system that allows everyone to share, manage and use data. There are many types of databases. A relational database is a type of database. It uses a structure that allows us to identify and access data in relation to another piece of data within the database. Data in a relational database is organized into tables.

Relational database design (RDD) models information and data into a set of tables with rows and columns. Each row of a relation/table represents a record, and each column represents an attribute of data. The Structured Query Language (SQL) is used to manipulate relational databases.

Rationale

Ease of Data Retrieval

Relational database accessing data does not require navigating a pathway through a tree or hierarchy, User can just use query to simply retrieve data from any table. If users require multi tables then just use join to include all the information.

Normalization

Normalization can help the database reduce redundant data, with more tables it will reduce the index per table, so that memory usage is less and data search/modification is faster and easier. For example, if you want to delete one item, every data reference to that will be automatically deleted also, you do not need to go to another table to delete the attribute that contains the same item.

Flexibility

the realtional data modeling will make the database easy to modifi and maintain, when you want to add a new data or add a new structure it is esay and will not affect other data, when you want to delete a item, you do not need to delete one by one that item related to that delete, when you delete it, it will automatically delete every related item.

Activities and Artifacts

Define relations/attributes

Find the relationship between each attribute. In other word, find the property of each table. For example, a Bank user will have account, password, name, date of birth, address, phone, email.

Define primary keys

find out the unique attribute for each table, in order to identify each column, usually is id.In the Bank user example Primary keys will be the account for each user.

Define relationships

Define each table's relationship, one to one, one to many, many to many. For example, a bank user can have many bank acount so user table will have one to many relationship with bank acount table.

Normalization

To Normalize each table confirms there are no multiple values in each column of the table. It makes sure each attribute is dependent on the primary key, if not separates it into another table. Confirms there are no redundancies in the table. For example, in personal information, a country will be separated into a country table because many people may have the same country.

Tools

Lucidchart

Lucidchart is cloud-based and a collaborative diagram software. Free plan comes with a limit of 3 diagrams and 60 objects per diagram.

dbdiagram.io

dbdiagram.io is a quick and simple database designer to help you draw your database diagrams using their own Domain-specific language (DSL). They have a simple language to define, you can easily edit/copy without leaving the keyboard. \ They focus exclusively on drawing database relationship diagrams. Allows you to save and share your diagrams

SQLDBM

SQLDBM offers you an easy, convenient way to design your database absolutely anywhere on any browser, working away without need for any extra database engine or database modelling tools or apps. They incorporate any needed database rules and objects such as database keys, schemas, indexes, column constraints and relationships. Only support MySQL and MS SQL.

QuickDBD

QuickDBDhelps you quickly draw a diagram using just text input. Free for 1 public diagram and 10 tables, plans at $14/month for unlimited tables and private diagrams. It quickly captures ideas, and you can draw schema diagrams using only the keyboard. Supports exporting to SQL, PDF and Word.
