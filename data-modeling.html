Relational Database Design

  • Skill Classification:Hard Technical Skill
  • Related Software Engineering Area(s) : Software Design

A Database is a system that allows everyone to share, manage and use data. There are many type of database. A relational database is a type of database. It uses a structure that allows us to identify and access data in relation to another piece of data in the database. Data in a relational database is organized into tables.

Relational database design (RDD) models information and data into a set of tables with rows and columns. Each row of a relation/table represents a record, and each column represents an attribute of data. The Structured Query Language (SQL) is used to manipulate relational databases

Activities and Artifacts

Define relations/attributesL

Find the relationship between each attribute. In other word, find the property of each table. For example, a Bank user will have account, password, name, date of birth, address, phone, email.

Define primary keys

find out the unique attribute for each table, in order to identify each column, usually is id.In the Bank user example Primary keys will be the account for each user.

Define relationships

Define each table's relationship, one to one, one to many, many to many. For example, a bank user can have many bank acount so user table will have one to many relationship with bank acount table.

Normalization

Normalize each table make sure there are no multiple value in each column of the table. Make sure each attribute is depend on the primary key, if not seprate into another table. make sure there are no redundant in the table, for, example, In personal inforamtion country will be seprate into country table becasue many person may has the same country.

Tools

Lucidchart

Lucidchart is cloud-based and collaborative diagram software Free plan comes with a limit of 3 diagrams and 60 objects per diagram

dbdiagram.io

dbdiagram.io is a quick and simple database designer to help you draw your database diagrams using their own Domain-specific language (DSL). They have a simple language to define, you can easily edit/copy without leaving keyboard. They focus exclusively on drawing database relationship diagrams. Allow you to save and share your diagrams online

SQLDBM

SQLDBM offers you an easy, convenient way to design your database absolutely anywhere on any browser, working away without need for any extra database engine or database modelling tools or apps. They incorporates any needed database rules and objects such as database keys, schemas, indexes, column constraints and relationships. Only support MySQL and MS SQL.

QuickDBD

QuickDBDhelps you quickly draw a diagram using just text input. Free for 1 public diagram and 10 tables, plans at $14/month for unlimited tables and private diagrams. It quickly captures ideas, and you can draw schema diagrams using only the keyboard. Support exporting to SQL, PDF and Word.
