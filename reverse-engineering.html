Reverse Engineering

  • Skill Classification: Hard Skill
  • Related Software Engineering Area(s) : Software Maintenance

Reverse engineering, also called back engineering, is the process by which a man-made object is deconstructed to reveal its designs, architecture, code or to extract knowledge from the object. Reverse engineering is a process of examination only; it does not change the software or result in new software. Reverse engineering is the process of analyzing software to identify the softwareâ€™s components and their inter-relationships and to create representations of the software in another form or at higher levels of abstraction.

The reverse engineering process, as such, is not concerned with creating a copy or changing the artifact in some way; it is only an analysis in order to deduce design features from products with little or no additional knowledge about the procedures involved in their original production. Reverse engineering may also be used to create interoperable products. Reverse engineering efforts produce call graphs and control flow graphs from source code.There are two components in reverse engineering: redocumentation and design recovery. Redocumentation is the creation of new representation of the computer code so that it is easier to understand. Meanwhile, design recovery is the use of deduction or reasoning from general knowledge or personal experience of the product in order to fully understand the product functionality

Software reverse engineering can help to improve the understanding of the underlying source code for the maintenance and improvement of the software, relevant information can be extracted in order to make a decision for software development and graphical representations of the code can provide alternate views regarding the source code, which can help to detect and fix a software bug or vulnerability. Frequently, as some software develops, its design information and improvements are often lost over time, but this lost information can usually be recovered with reverse engineering. This process can also help to cut down the time required to understand the source code, reducing the overall cost of the software development. Reverse engineering can also help to detect and eliminate a malicious code written to the software with better code detectors. Reversing a source code can be used to find alternate uses of the source code, such as to detect unauthorized replication of the source code where it wasn't intended to be used, or to reveal how a competitors product was built. Reverse engineering techniques are often used to find vulnerabilities in an operating system (OS), in order build a computer virus that can exploit the system vulnerabilities.

Prerequisites

One must possess the below listed skills before starting with reverse-engineering:

  • Solid knowledge of the underlying platform APIs
  • Common design patterns and general architectural concepts
  • Knowledge of call graphs and control flow graphs
  • Understanding of the code to be reverse-engineered and understanding of working of Operating Systems work
  • Availability of Original Design of Product
  • Availability of Source Code for Software
  • Knowledge of reverse engineering/debugging tools
  • Life-cycle analysis and Cost/benefit analysis should be conducted
  • Concepts relevant to a particular technology

Rationale

No/Outdated Software Documentation

Software must have up to date software documentation. But when higher-level aspects of the program perhaps poorly documented or documented but no longer valid, are discovered, it has no way of understanding how it works. This means code logic needs to be studied and the system needs to be reverse-engineered.

Interface Systems

A system is required to interface to another system and how both systems would negotiate is to be established. Such requirements typically exist for interoperability.

Discovery of product vulnerabilities

Reverse engineering enables finding faults in the product. This is to ensure the safety and well-being of the productâ€™s users. It is best for an issue to arise during the research phase rather than the distribution phase.

Exploring existing designs and maneuvers

Reverse engineering enables to see what already exists. This includes any parts, structures, or processes that could benefit communities in other ways. Examining current products leads to innovation and discovery, all of this is possible due to reverse engineering.

Roles

Reverse Engineering Software Developer

Reverse Engineering Software Developer is engaged in reverse engineering the software applications, understand the use of network protocols, file formats and software libraries in applications as well as development of digital signatures for network traffic-based attack detection.

Senior Software Reverse Engineering Analyst

Software Reverse Engineering Analyst participates in major projects and campaigns, and provide software analysis, patent assessments and software reverse engineering, documentation and patent portfolio development.

Reverse Engineering Malware Specialist

The most prominent role of Reverse Engineering Malware Specialist is to check and be aware of any suspicious computer behavior. Their working sphere consists of examining, identifying, and understanding the nature of cyber-threats. These can range from viruses, worms, bots, rootkits, to Trojan horses.

Software Reverse Engineer

Software Reverse Engineer is responsible to study how the program performs certain operations, to improve the performance of a program, to fix a bug, to identify malicious content in a program such as a virus or to adapt a program written for use with one microprocessor for use with another.

Activities and Artifacts

Analysis of Product Attributes

Measurement of product attributes is a key component since they affect the productâ€™s performance capacity. 3D scanning technologies are used make these kinds of measurements. With three-dimensional scanners, engineers can gain accurate readings of the productâ€™s specs and have this information automatically logged in their databases.

Creation of CAD Drawings

After all the pertinent information has been gathered and recorded, this data to create computer-aided design (CAD) drawings for subsequent analysis and development. CAD drawings are digital two-dimensional and three-dimensional representations of the products, which you can use to analyze the productâ€™s design. These digital models help to unveil design intent and inform the creation of a reverse-engineered component.

Deconstructing Individual Components

It is not uncommon that there is limited knowledge about the engineering methods that went into creating the product. Therefore, the challenge is to gain a working knowledge of the original design by disassembling the product piece-by-piece or layer-by-layer which enables to determine how a part was designed so that it can be recreated.

Rediscover Formulas

If an old piece of computer equipment had functions that have since been lost amid the subsequent changes in technology, reverse engineering allows manufacturers to rediscover these formulas and bring them up to date. Reverse engineering also enables you to develop components that bridge the new and the old, allowing users of older equipment to connect their devices to modern computing equipment.

Recreate Design of Legacy Products

Every once in a while, one of the machine parts will wear out and the preferable option is to retain the same equipment and replace the faulty part. Reverse engineering involves activities to regain design data on the long-discontinued products manufactured before the days of computer-aided design and digital file storage, mainly based on long-lost paper blueprints. One can use a 3D scanner to digitally replicate the design of the defective part. From there, a new copy of the component can be created, installed into the machine and create archives of the product legacy.

Product Security Analysis

To examine how a product works by determining the specifications of its components and estimate costs, identifying potential patent infringement. Also it involves acquiring sensitive data by disassembling and analyzing the design of a system component. Another intent may be to remove copy protection, or circumvention of access restrictions.

Disassembling and Reassembling Components

During disassembly, the team removes each part from the board, one after another. The parts must be set aside for safekeeping and organized in the order they were removed. This way, the board can be reassembled later on, if necessary. In cases where the objective is to replicate the design in question, engineers might test their ability to replicate the product by disassembling and reassembling an existing copy several times over.

Redocumentation

All the replicated products or recreated designs should be well documented with proper description of process, approaches followed and relevant details that can serve helpful for future purposes.

Real-World Example/Scenario

Phoenix BIOS

In mid-1980s, San Jose-based Phoenix Technologies Ltd. reverse-engineered IBM PC's proprietary BIOS using what's called a "clean room" or "Chinese wall" approach to protect against charges of having simply (and illegally) copied IBM's BIOS. The resulting Phoenix BIOS was different from the IBM code, but for all intents and purposes, it operated identically and there was no copyright infringement.

Bell Laboratories

In 1987 Bell Laboratories reverse-engineered the Mac OS System 4.1, originally running on the Apple Macintosh SE, so they could run it on RISC machines of their own.

Role of Academia/Industry

Academia: Students engaged in computer science and software engineering programs typically lack experience and understanding of the multidisciplinary complexity of software engineering projects. Hence, the software engineering curriculums are designed in order to impart knowledge pertaining to various processes, concepts and stages of reverse engineering. Majority of the academic projects focus on improving the problem solving and analytical skills, enhance the ability to develop and innovate, develop and design the proof-of-concepts. Honing of all these skills can lead the students to become a good reverse engineer. There are certain training programs specially organized to train the student for creating computer-aided design drawing that provides exposure to various software used for creating 3D images, one of the important activity involved for reverse engineering. Further, the software academic projects expects the students to prepare a proper documentation for application or software developed. Thus, it can be concluded that the software projects in academia are aligned with the knowledge required by a reverse engineer in industry.

Industry:The tools required for debugging that are developed in industry have now been used by students in academic projects to gain hands-on knowledge. Vulnerability analysis should be conducted correctly because mistakes in identifying bugs or virus can lead to dangerous consequences and costly damages not only to the software product but also to the organization as a whole. Expert knowledge on threats, attacks, and incidents practiced at industrial level is utilized to train the students for cryptanalysis and implement symmetric key algorithms for various academic software projects. The steps implemented as a part of reverse engineering process in industry are then formulated and included in the curriculum so that the students can get exposure and awareness of trends and practices followed in industry while reverse engineering the products from implementation phase to analysis phase


Tools

IDA Pro

It is an interactive disassembler, which is widely used for software reversing. It has the in-built command language (IDC), supports a number of executables formats for variety of processors and operating systems. Also it has a great number of plugins which allow to extend the disassembler functionality even further like Hex Rays Decompiler, Lighthouse and so on.

.NET Reflector

It enables to quickly and efficiently take a peek at application's structure and code, recreating of the whole project for Visual Studio, from decompiled application and integration with Microsoft Visual Studio allows for simultaneous debugging of own code and code of closed libraries.

Java Decompiler

JD-GUI or Java Decompiler is a decompiler for Java applications. It allows for viewing of compiled units code*.class or whole *.jar bundles. It contains very useful search engine with filters that allow for searching by names, types, constructors, fields, methods and text strings.

OllyDbg

It is a de facto a standard debugger for Windows in the world of reverse engineering. It has capabilities of application code analysis and allows for interference with almost every aspect of application running, conditional code tracing and has vast plugins database that allow for controlling of debugger running from the script level.
