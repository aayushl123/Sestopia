DEBUGGING

  • Skill Classification: Technical (Hard)
  • Related Software Engineering Area(s) : Software Construction, Software Testing

What is this skill about?
Debugging as a skill is the capability of fixing a bug in the software. Any sort of existing or potential errors in the software is assumed to be a bug. Debugging can be defined as the identifying, analysing and removing errors. This activity begins after the software fails to execute properly and concludes by solving the problem and successfully testing the software again.

"Programming allows you to think about thinking, and while debugging you learn learning." - Nicholas Negroponte

It is believed that the success of a programmer and his expertise is not in just writing the code but lies in their ability to debug accurately.Debugging entails determining the faulty lines of code and replacing them so that they behave as per the requirements elicited and expected. So, the process of debugging is the composition of four stages, commencing with reasoning the code, followed by hypothesizing about what might be wrong, testing the hypothesis, and consequently making the required modifications.
Around 60 percent of the software development activity goes into debugging and accordingly, a plethora of strategies, techniques, guidelines, and standard practices are available for debugging. Although debugging plays a vital role in making a product reliable, unfortunately it has been the most overlooked activity among software developers.

Understanding the thin line between Testing and Debugging
There exists a huge misconception about testing and debugging. Both words are at times used interchangeably which is a significant blunder.
Testing aims at finding a problem while Debugging aims at solving the problem. Testing focuses on finding bugs, errors, etc whereas debugging starts after a bug has been identified in the software.Only after the testing team reports the defect, debugging can take place. With debugging, the developer identifies the problem in the system/application/code Testing can be manual or automated but the latter has to be manual. Debugging requires a lot of knowledge, skills, and expertise. It can be supported by some automated tools available but is more of a manual process as every bug is different and requires a different technique, unlike a pre-defined testing mechanism.So, in layman language we could say that debugging and testing are complementary processes. The purpose of testing is to identify what happens when there is a fault in a program's execution of source code. The purpose of debugging is to locate and fix the fault.

Rationale

In the software development process

Debugging is an important aspect in the software development cycle. It’s a mandatory consequence of the testing phase, extremely necessary for fixing all the problems that were observed as potential errors in the testing phase. Thus the rationale for debugging in this case is to fix the faulty code flow and find a optimized solution to get the halted system flow, running.

In the industrial scenario

There have been many instances where lack of effectiveness in the company’s debugging workflow, has led to the loss of millions of dollars. Companies need the most reliable product to reduce any long-term costs and a bug free product is the only solution for it. Hence, debugging proves to be a necessity especially in the industrial world.

Roles

Debug Engineer

A Debug Engineer has an impact on the tools and workflows used by developers every day to launch, debug, test, and analyze the software that they build for different platforms and shapes the software’s deep integration throughout the software stack.

System Debug Lead

A System Debug Lead leverages debug features to accelerate testing and improve test effectiveness. Debugs and performs root cause determination of improper or inconsistent operation.

Software Analyst (Debugging)

A Software Analyst analysis the source code for potential error flow, develops and maintains a portfolio of debug tools and libraries that are hosted and targets the potential bugs in OSes including Windows, Linux, and other embedded system real time OSes (RTOS).

Relation Engineer - Debugging

A Relation Engineer debugs customer code, writes code snippets for customers as well as continuously refines and documentes on boarding processes while troubleshooting.

Activities and Artifacts

Related Activities

1. Problem analysis, identification and report generation.
2. Communicating the report details to the software developer for the verification of bugs found during source code analysis.
3. Creating log files using a standard log library and keeping track of the logs of every execution by setting up breakpoints.
4. Error Analysis using modeling, documentations, finding and testing candidate flaws, etc.
5. Error Resolution by performing necessary changes to the code after proper validation of the solution decided.
6. Validation of corrections i.e. solution implementation.
7. Implementation of strategies such as code review, unit tests and pair programming.

Artifacts

1. Log files
2. Error reports
3. Solution documentation

Debugging Strategies

1. Static debugging - the developer examines the code without executing the program.
2. Print debugging - the developer watches recorded print statements and monitors flow.
3. Remote debugging - the developer's debugger runs on a different system than the program that is being debugged.
4. Post-mortem debugging - the developer only stops to debug the program if it experiences fatal exceptions.

Debugging Techniques

1. Brute Force Method - The program is loaded with print statements to print the intermediate values with the hope that a number of the written values can facilitate to spot the statement in error. A Source Code Debugger is used for sytematizing this aproach
2. Cause Elimination method - In this approach, several causes presumed to be the reason of failure are noted and then elimination tests are conducted for each of the cause.
3. Backtracking - This approach involves derivation or say tracking of the source code from the point of error symptom till yhe point of error.
4. Program Slicing - This approach has similar characteristics to the approach of backtracking. Here the search house is decreased by process slices.

Role of Academia/Industry

There are many programming related undergraduate and graduate courses as a part of our academics that enhance the skill of debugging. Many academic projects require students to analyse the problem, identify it and prepare proper solution documentation, which are prerequisites of improving debugging skill. Also strategies such as backward analysis of the problems. forward analysis of the problem as well as defect analysis which are a part of the curriculum as a mandatory procedure for coding base projects, also aid in cultivating the skill.

The industrial workflow and development process also cultivates this skill. Any software development lifecycle- SDLC, requires a testing phase. Moreover, debugging is a necessary consequence of this phase for solving any potential errors found. Furthermore, industrial working process necessitates that debugging should be implemented in various steps, using standardized strategies and implementing standard techniques while having a similar cognition with the developer as well as the tester. This improves the skill of debugging to a large extent.


Tools

Chrome DevTools

The Google Chrome Developer Tools, also known as Chrome DevTools, are web authoring and debugging tools built right into the browser. They provide developers deeper access into their web applications and the browser. Everything from testing the viewport on a mobile device to debugging a website on the fly, and even measuring the performance of an entire website or individual assets can be easily done using it.

VisualCode Debugger

One of the key features of Visual Studio Code is its great debugging support. VS Code's built-in debugger helps accelerate your edit, compile and debug loop.VS Code has built-in debugging support for the Node.js runtime and can debug JavaScript, TypeScript, or any other language that gets transpiled to JavaScript. For debugging other languages many extensions are provided.

IBM Rational S/W Analyzer

IBM® Rational® Software Analyzer is an extensible software development solution that enables static analysis code reviews and problem identification very early in the development cycle. The costs associated with fixing problems increase as they mature through development. IBM Rational Software Analyzer is designed to reduce this expense by identifying code-level defects earlier in the software development process.
